¿Qué son los condicionales?

Los condicionales permiten que un programa ejecute diferentes bloques de código según se cumpla o no una condición. Son la base de la toma de decisiones en programación.

Operadores de comparación

  • == → igual a (ej: 5 == 5 → True)
  • != → distinto de (ej: 5 != 3 → True)
  • > → mayor que (ej: 7 > 2 → True)
  • < → menor que (ej: 3 < 10 → True)
  • >= → mayor o igual (ej: 5 >= 5 → True)
  • <= → menor o igual (ej: 4 <= 6 → True)

Operadores lógicos

  • and → ambas condiciones deben ser True
  • or → al menos una condición debe ser True
  • not → invierte el valor lógico

If

Comprobar una condición: if evalúa una expresión booleana y, si es True, ejecuta su bloque.
Para qué sirve: activar lógica solo cuando se cumple un requisito.

edad = 18
if edad >= 18:
    print("Sos mayor de edad")
if condición:
    # bloque si la condición es True

Else

Cubrir el caso contrario: else define un bloque alternativo cuando la condición previa resultó False.
Para qué sirve: manejar el “si no” de forma explícita.

edad = 16
if edad >= 18:
    print("Mayor de edad")
else:
    print("Menor de edad")
if condición:
    # bloque si True
else:
    # bloque si False

Elif

Múltiples caminos en orden: elif (else if) permite evaluar una nueva condición cuando la anterior fue False.
Para qué sirve: encadenar alternativas ordenadas sin anidar varios if.

nota = 7
if nota >= 9:
    print("Excelente")
elif nota >= 6:
    print("Aprobado")
else:
    print("Desaprobado")
if condición1:
    # bloque1
elif condición2:
    # bloque2
else:
    # bloque3 (opcional)

Patrones típicos y buenas prácticas

Los condicionales no solo sirven para comparar números. También se usan para validar entradas, clasificar datos, controlar flujos y evitar errores. Aquí tenés ejemplos prácticos y consejos para escribir condicionales claros y eficientes.

1) Rango ordenado con elif (clasificación por intervalos)

temperatura = 12 if temperatura < 0: print("Bajo cero") elif temperatura < 15: print("Frío") elif temperatura < 25: print("Templado") else: print("Calor")
Resultado: Frío

Explicación: El programa evalúa la variable temperatura en orden. Como 12 no es menor que 0 pero sí es menor que 15, se ejecuta el bloque correspondiente y muestra “Frío”.

2) Validación por pertenencia (in / not in)

frutas = ["manzana", "banana", "pera"] if "kiwi" in frutas: print("Fruta válida") else: print("Fruta no encontrada")
Resultado: Fruta no encontrada

Explicación: Se comprueba si la cadena "kiwi" está dentro de la lista frutas. Como no está, se ejecuta el bloque del else.

3) Combinación lógica (and / or)

edad = 17 permiso = True if edad >= 18 and permiso: print("Acceso permitido") elif edad >= 18 or permiso: print("Acceso parcial") else: print("Acceso denegado")
Resultado: Acceso parcial

Explicación: La primera condición (edad >= 18 and permiso) es falsa porque la edad es 17. La segunda (edad >= 18 or permiso) es verdadera porque aunque la edad no cumple, permiso es True. Por eso imprime “Acceso parcial”.

4) Operador ternario (condición en una sola línea)

estado = "enviado" print("Enviado" if estado == "enviado" else "Pendiente")
Resultado: Enviado

Explicación: El operador ternario permite escribir un if/else en una sola línea. Como estado es igual a “enviado”, se imprime “Enviado”.

Comparaciones encadenadas

nota = 8 if 6 <= nota <= 9: print("Aprobado con buena nota")
Python permite encadenar comparaciones. La condición 6 <= nota <= 9 significa "nota mayor o igual a 6 y menor o igual a 9". Como nota es 8, se imprime el mensaje.

5) Negación (not)

usuario = "" if not usuario: print("El nombre de usuario no puede estar vacío")
Resultado: El nombre de usuario no puede estar vacío

Explicación: En Python, una cadena vacía se evalúa como False. Al aplicar not, se convierte en True, por lo que se ejecuta el bloque y muestra el mensaje de error.

6) Prevenir errores comunes (división por cero)

divisor = 0 if divisor != 0: print(10 / divisor) else: print("Error: no se puede dividir por cero")
Resultado: Error: no se puede dividir por cero

Explicación: Antes de dividir, se comprueba que el divisor no sea cero. Como sí lo es, se ejecuta el else y se evita un error en el programa.

7) Comparaciones encadenadas

nota = 8 if 6 <= nota < 9: print("Aprobado con buena nota")
Resultado: Aprobado con buena nota

Explicación: Python permite encadenar comparaciones. La condición 6 <= nota < 9 significa “nota mayor o igual a 6 y menor que 9”. Como nota es 8, se cumple y se imprime el mensaje.

  • Orden lógico: colocá las condiciones más específicas primero y las generales al final.
  • Un único flujo: en una cadena if/elif/else solo se ejecuta un bloque.
  • Claridad: usá nombres de variables descriptivos y condiciones fáciles de leer.
  • Evitar anidación profunda: preferí elif en lugar de varios if dentro de otros.
  • Validar entradas: comprobá que los datos sean correctos antes de usarlos.
  • Prevenir errores: usá condicionales para evitar operaciones inválidas (ej. división por cero).
  • Expresiones compactas: podés combinar comparaciones (ej: if 0 <= x <= 10:).
  • Legibilidad sobre trucos: es mejor un condicional claro que uno demasiado “ingenioso” pero confuso.

Resumen

  • Usá: if, elif, else según la lógica.
  • Combiná: operadores de comparación y lógicos.
  • Practicá: pertenencia (in), negación (not), ternario y comparaciones encadenadas.
